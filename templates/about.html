HYBRID SECURE

Encrypted File Storage Using Hybrid Cryptography



Objective: The aim is to establish a trustworthy platform for users to securely store their files on the cloud using a combination of hybrid cryptography techniques.

Methodology:

  1. Uploading the desired file onto the server, regardless of its format.
  2. Partitioning the uploaded file into N segments, based on a predefined fixed block size.
  3. Employing various selected encryption algorithms to individually encrypt these subsections. (The algorithm changes sequentially in a round-robin manner for each segment.)
  4. Storing the encryption keys for these algorithms within a designated file. Subsequently, this resulting file undergoes an additional layer of security through an alternative cryptographic algorithm. The user receives the key for this algorithm as a public key, which must be transmitted securely to the intended recipient. (Currently, symmetric encryption is utilized, with potential plans to transition to authentication and asymmetric encryption in the future.)

Restoration Process:


  1. Uploading the key file (.pem) to the server, a request made to the sender.
  2. Deciphering the keys associated with the cryptographic algorithms.
  3. Decrypting all N subsections using the identical algorithms employed during the encryption phase.
  4. Reassembling the N subsections to reconstruct the original file, which is then made available for user download.
